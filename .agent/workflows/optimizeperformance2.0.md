# Optimize Performance 2.0

## 1. Lazy Loading Below-the-Fold Components
To reduce the initial JavaScript bundle size and improve the initial load metrics (LCP/FCP), use `React.lazy` and `Suspense` for components that are not immediately visible.

- Identify heavy components that are rendered below the initial viewport.
- Replace standard imports with lazy imports: 
  ```tsx
  const ComponentName = React.lazy(() => import('./path/to/Component'));
  ```
- Wrap the components in a `<Suspense>` component with a lightweight fallback (e.g., an empty `div` with a minimum height matching the component's expected height to prevent layout shifts).

## 2. DOM Pruning with content-visibility
For components with exceptionally deep or complex DOM structures (e.g., those generated by Figma exports) that might be prone to breaking if manually flattened, use the `content-visibility` CSS property. This skips the rendering and layout calculations for off-screen elements until they approach the viewport, significantly reducing Total Blocking Time (TBT).

- Define a CSS utility class:
  ```css
  .content-visibility-auto {
      content-visibility: auto;
      contain-intrinsic-size: auto 800px; /* Adjust height as an estimate of the rendered component */
  }
  ```
- Apply this `.content-visibility-auto` class to the wrapper elements of heavy, below-the-fold sections.

## 3. Eliminating Animation TBT (Forced Reflows)
Continuous animations on layout properties (like `left`, `top`, `width`, `height`) cause forced synchronous layouts on the main thread, resulting in high TBT.

- Identify components running continuous loops or transitions on position/layout properties.
- Refactor these animations to use hardware-accelerated transforms: `transform: translate3d(x, y, 0)`.
- This pushes the animation calculation to the GPU, entirely avoiding layout blocks on the main browser thread.

## 4. LCP and Asset Delivery Optimization
Ensure that the browser prioritizes loading the most critical assets for the initial view while deferring heavy non-critical media.

- **Prioritize Hero Media:** Use `fetchPriority="high"` on the largest above-the-fold image (the LCP element).
- **Defer Background Media:** If using a background `<video>`, change its `preload` attribute from `"auto"` to `"metadata"` to stop the browser from aggressively downloading large media chunks during initial page load, which otherwise competes with critical layout scripts.
